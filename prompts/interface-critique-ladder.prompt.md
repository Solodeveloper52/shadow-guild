# interface-critique-ladder

Given UI context and screenshots/code, produce:

- Critical usability blockers
- Information hierarchy weaknesses
- Interaction and accessibility issues
- Visual consistency and design-system drift
- Prioritized fix ladder (now, next, later)

## Deep Prompt Contract

### Required Inputs

- Objective, constraints, baseline evidence, and audience.

### Required Outputs

- Prioritized summary, detailed analysis, risks, and next actions.

### Quality Rules

- Label assumptions, tie recommendations to evidence, include fallback.
